Memory is shown in mebibytes (1024 * 1024 bytes.)

YJIT memory is divided into normal memory and memory for inline/outlined generated code. By default YJIT allocates 128MB each for inline and outlined code. The amount can be tuned using a command-line parameter. Code sizes are always rounded upward - even a single byte is counted as a full mebibyte since exceeding the limit will cause a crash.

Memory usage other than code sizes is rounded down, as normal.
